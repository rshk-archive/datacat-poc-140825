datacat.utils.archives¶

Abstraction layer to access different kinds of file archives using a common interface.

Note

Currently the functionality is limited to the one required for the data extraction pipelines, i.e. file listing and retrieving.

In the future, this module might be split into its own package and more functionality added (eg. write support, ...).

datacat.utils.archives.open_archive(filename)¶

Opens an archive using the appropriate opener depending on the type.

Strategy:

Try opening with the most appropriate opener for a file with the given extension, then fallback trying with others.
Parameters:filename – Name of the file to be opened as an archive

class datacat.utils.archives.BaseArchive(filename)¶

Base interface for the archive abstraction.

__init__(filename)¶

Initialize things. This method should also already open the archive, in order to check whether it is a supported format or not. If not, it should raise an ArchiveOpenFailure exception, to tell open_archive() to try with the next opener.

__iter__()¶

Iterate over archive members, yielding BaseArchivedFile instances for each one.
