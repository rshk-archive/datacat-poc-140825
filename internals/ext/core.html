datacat.ext.core¶

datacat.ext.core.core_plugin = datacat.ext.base.Plugin('datacat.ext.core')¶

The core plugin, providing most of the “standard” functionality. Having the core functionality in a plugins allows users to easily extend and replace it to fit any custom needs.

datacat.ext.core.make_dataset_metadata(dataset_id, config, metadata)¶
Hook:make_dataset_metadata
datacat.ext.core.get_dataset_index(*a, **kw)¶

API view returning a (paged) list of datasets.

HTTP url:/data/
HTTP methods:GET

The view returns a list of dictionaries representing dataset objects. The schema is entirely up to the enabled plugins; the core implementation only guarantees that the id field is set to the correct dataset id.

Example request:

GET /api/1/data/ HTTP/1.0

Example response:

HTTP/1.0 200 OK
Content-type: application/json
Link: ?start=10&size=10; rel=next, ?start=50&size=10; rel=last
X-page-total: 60
X-page-start: 0
X-page-size: 10

[{"id": 1}, {"id": 2}, {"id": 3}, ..., {"id": 10}]
datacat.ext.core.get_dataset(*a, **kw)¶

API view returning information for a single dataset.

HTTP url:/data/<dataset_id>
HTTP methods:GET

Example request:

GET /api/1/data/1 HTTP/1.0

Example response:

HTTP/1.0 200 OK
Content-type: application/json

{"id": 1, ...}
