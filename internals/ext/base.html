datacat.ext.base¶

class datacat.ext.base.Plugin(import_name)¶
__init__(import_name)¶
Parameters:import_name –

Usually the name of the module concatenated with the plugin name, eg:

myplugin = Plugin(__name__ + '.myplugin')
setup(app)¶

Setup the plugin by attaching an application

install()¶

Method called to install the plugin, i.e. the first time it appears in the PLUGINS list.

uninstall()¶

Method called to completely uninstall a plugin.

It should be called via some administration CLI command.

enable()¶

Method called when a plugin gets enabled, i.e. when it appears in the PLUGINS list after it was missing.

disable()¶

Method called when a plugin gets disabled, i.e. when it disappears from the PLUGINS list.

upgrade()¶

Perform necessary database schema upgrades.

hook(hook_type)¶

Decorator function to register a “hook” function, to be used later for various purposes.

Parameters:hook_type (str, list, tuple) – A string (or list/tuple of) indicating the hook(s) to which the handler should be attached
call_hook(hook_type, *a, **kw)¶

Helper function to call all the handlers for a given hook type in this plugin.

Extra arguments will be passed directly to the hook handler(s)

Parameters:hook_type – The hook type
Returns:a list of return values for all the called handlers

route(*a, **kw)¶

Decorator function to register an API view for this plugin.

The underlying Blueprint.route() method will be called.

Blueprints are then registered under the /api/1/ prefix.

task(*a, **kw)¶

Decorator function to register a celery task.

Note

Tasks should be registered with their full namespace, eg:

@myplugin.task(name=__name__ + '.mytask')
def mytask(foo, bar):
    pass
